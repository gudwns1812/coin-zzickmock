# UI Foundations

## 목적

이 문서는 `coin-zzickmock` 프론트엔드의 가장 바닥에 있는 시각 언어를 정의한다.
새 화면을 만들기 전에 먼저 이 문서를 기준선으로 삼는다.

## 제품 인상

이 프론트엔드는 다음 인상을 유지해야 한다.

- 데스크톱 중심의 넓고 안정적인 금융 대시보드
- 흰 카드 위에 정보를 정리하고, 바깥은 밝은 블루 그라데이션으로 감싼 구조
- 숫자와 상태 변화가 먼저 읽히는 정보 우선형 인터페이스
- 지나치게 차갑지 않도록 약한 모션과 그라데이션으로 리듬을 준 화면
- "앱"보다는 "워크스테이션"에 가까운 밀도 있는 업무 화면

## 핵심 원칙

- 새 화면은 먼저 "얼마나 빨리 읽히는가"로 평가한다.
- 장식은 허용하지만 데이터 가독성을 가리면 안 된다.
- 더 화려해질 수는 있어도 더 산만해지면 안 된다.
- 사용자가 자주 보는 숫자, 등락, CTA는 항상 비슷한 위치와 무게를 가져야 한다.

## 토큰 기준

토큰의 출처는 `frontend/app/globals.css`다.
새 UI는 임의 값보다 기존 토큰을 우선 사용한다.

### 색상

- 브랜드/기본 강조: `--color-main-blue`
- 경고/하락/파괴적 액션: `--color-main-red`
- 중립 보더/배경선: `--color-main-light-gray`
- 기본 보조 텍스트: `--color-main-dark-gray`
- 약한 보조 텍스트: `--color-sub`
- 보조 강조: `--color-keyword`

금융 도메인 의미 체계:

- 상승은 녹색
- 하락은 빨강
- 중립은 회색

이 규칙은 차트, 배지, 가격 텍스트, 상태 라벨 전체에서 유지한다.

### 간격

- 기본 단위는 `--spacing-main`
- 반복 레이아웃 간격은 `gap-main`, `gap-main-2`, `gap-main-3`
- 카드 패딩은 `p-main` 또는 `p-main-2`
- 작은 액션 버튼은 `px-main py-1` 계열

### 반경과 그림자

- 기본 반경: `rounded-main`
- 카드 그림자는 얕고 정보 패널에 가까운 무게를 유지한다.
- 기본 그림자 계층은 `shadow-sm`, `shadow-md`, `shadow-color`

### 타이포그래피

기본 서체는 `Pretendard Variable`이다.

- 본문: `text-base-custom`
- 보조 설명: `text-sm-custom`
- 작은 레이블: `text-xs-custom`
- 섹션 제목: `text-xl-custom`, `text-2xl-custom`
- 강조 수치/헤드라인: `text-2xl-custom` 이상

새 화면은 임의 `text-[14px]`, `text-[15px]`를 늘리기보다 이 스케일을 먼저 검토한다.

## 시각적으로 허용하는 것

- 블루 중심 브랜드 강조
- 보조 퍼플 그라데이션
- 흰 카드와 얕은 그림자
- 원형 아이콘 배경
- 금융 지표 중심의 데이터 카드
- 별 배경, 숫자 카운팅 같은 제한적 모션

## 시각적으로 피해야 하는 것

- 과한 glassmorphism
- 어두운 배경 위 네온 중심 테마
- 카드마다 다른 반경/그림자/패딩 체계
- 텍스트보다 장식이 먼저 보이는 hero-first 구성
- 화면마다 새로운 색 의미를 만드는 것

# Data Display

## 목적

이 문서는 카드, 차트, 숫자, 리스트 같은 데이터 표시 방식을 정의한다.

## 카드 시스템

이 프로젝트의 기본 UI 단위는 카드다.

주요 카드 유형:

- 차트 카드
- 목록 카드
- 요약 카드 (상단 지표용: 아이콘, 타이틀, 메인 수치, 보조 수치의 수직 구조)
- 액션 카드

공통 규칙:

- 기본 배경은 `bg-white`
- 기본 반경은 `rounded-main`
- 카드 구분은 얕은 그림자나 옅은 보더를 우선 사용
- 제목, 본문, 보조 액션의 계층이 분명해야 한다

권장 카드 구조:

1. 헤더: 제목, 짧은 설명, 우측 액션
2. 본문: 차트, 수치, 목록
3. 보조 정보: 범례, 기간 전환, 상태 메시지

카드 제목 규칙:

- 최상위 분석 카드 제목은 그라데이션 텍스트 사용 가능
- 일반 카드 제목은 진한 텍스트를 기본으로 한다
- 한 화면에서 그라데이션 제목은 1~3개 정도로 제한한다

## 차트와 숫자

차트는 장식보다 추세 인지에 집중한다.

규칙:

- 차트 종류가 달라도 색 의미는 동일해야 한다.
- 상승/양수는 녹색, 하락/음수는 빨강, 중립은 회색
- 큰 숫자는 천 단위 구분을 유지한다.
- 등락 정보는 변화량과 비율을 함께 보여 준다.

권장 패턴:

- 차트 아래나 옆에 핵심 요약 수치를 둔다.
- 가격 업데이트 시, 가격 컴포넌트의 **사각형 영역(Boundary)** 전체에 일시적인 배경색 플래시 효과를 적용한다 (상승 시 Green, 하락 시 Red). 효과는 수신 즉시 강조된 후 빠르게 페이드 아웃된다.
- 기간 전환 버튼은 차트 외곽 상단에 둔다.
- hover, tooltip, 보조 선은 읽기 보조 범위로 제한한다.
- 차트 바깥 장식 보더와 배경은 최소화한다.
- 포지션선과 주문선 같은 가로 보조선은 차트 안에서 가격 판단을 돕는 수준으로만 사용한다.

## 리스트와 종목 행

인기 종목, 검색 결과, 보유 종목 리스트는 공통 구조를 따른다.

- 좌측: 종목 이미지 또는 이니셜 fallback
- 중앙: 종목명, 코드, 현재가, 보조 정보
- 우측: 등락 또는 이동 affordance

행 규칙:

- hover 시 약한 블루 배경 또는 보더 강조를 준다.
- 클릭 가능 항목은 가능한 카드 전체를 누를 수 있게 한다.
- 이름과 코드, 가격과 등락은 서로 다른 시각 무게를 가져야 한다.
- 보조 아이콘은 기본 숨김 후 hover 노출을 허용한다.

## 숫자 표현 원칙

- 가격은 빠르게 스캔 가능해야 한다.
- 상태색만으로 의미를 끝내지 않고 수치와 기호를 함께 쓴다.
- 핵심 숫자는 보조 설명보다 항상 한 단계 더 굵고 크다.
- 등락률, 손익, 요약 값은 동일 화면 안에서 포맷이 흔들리지 않아야 한다.

## 데이터 표시 체크리스트

- 카드 제목만 보고도 섹션 목적이 드러나는가
- 숫자, 변화량, 비율이 함께 읽히는가
- 상승/하락 색 의미가 기존과 같은가
- fallback 이미지/빈 데이터 상태가 자연스러운가

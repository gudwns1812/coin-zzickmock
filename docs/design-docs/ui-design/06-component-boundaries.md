# Component Boundaries

## 목적

이 문서는 어떤 UI를 어디에 두고, 어떤 기준으로 shared 컴포넌트로 올릴지 정의한다.

## 현재 구조 해석

현재 폴더 구조에는 아래 레이어가 공존한다.

- `components/ui`
- `components/ui/shared`
- `components/router/*`
- `components/animate-ui/*`

디자인 관점의 역할:

- `components/ui`: 화면 전반에서 재사용 가능한 기본 블록
- `components/ui/shared`: 앱 크롬, 버튼, 입력, 헤더, 푸터, 드롭다운 같은 공통 요소
- `components/router/*`: 특정 라우트에 강하게 결합된 도메인 섹션
- `components/animate-ui/*`: 시각 강조용 보조 프리미티브

## 새 컴포넌트 배치 규칙

- 도메인 강한 카드나 섹션은 `components/router/<route>` 쪽에 둔다.
- 버튼, 입력, 모달, 상태 카드, 배지 같은 범용 요소는 `components/ui` 또는 `components/ui/shared`에 둔다.
- 페이지 전용 장식 컴포넌트는 공용 레이어에 올리지 않는다.
- CSS Module은 Tailwind로 설명하기 어려운 특수 형태에만 제한적으로 사용한다.

## shared 승격 기준

아래 중 두 가지 이상에 해당하면 shared 승격을 검토한다.

- 두 화면 이상에서 재사용된다
- 의미 체계가 동일하다
- 스타일 변경 시 한 곳에서 관리하는 편이 안전하다
- 접근성/상태 표현 규칙을 공통으로 강제해야 한다

우선 정리 대상:

- 카드 헤더 패턴
- 그라데이션 제목 패턴
- hover tint 버튼 패턴
- 검색 결과/종목 목록 행
- 에러/빈 상태/가이드 상태 패턴

## 하지 말아야 할 것

- 페이지 안에서 버튼/입력/모달을 새로 재정의하기
- 도메인별로 등락 색 의미를 바꾸기
- 카드 단위 대신 페이지 전체를 하나의 거대한 패널로 만들기
- 레거시 오탈자 폴더나 명명 패턴을 새 코드에 복제하기

## 운영 원칙

- 새 공통 패턴이 생기면 이 폴더 문서를 먼저 갱신한다.
- 규칙이 3번 이상 반복되면 shared 레이어 승격을 검토한다.
- 기존 톤에서 크게 벗어나는 시안은 어떤 기준을 의도적으로 벗어나는지 설명할 수 있어야 한다.

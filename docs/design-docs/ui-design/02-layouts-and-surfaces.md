# Layouts And Surfaces

## 목적

이 문서는 앱 프레임, 페이지 그리드, 표면 구조, 반응형 정책을 정의한다.

## 표면 구조

현재 서비스는 두 층의 표면 구조를 가진다.

- 앱 전체 배경: 밝은 블루 계열 그라데이션
- 실제 콘텐츠 표면: 흰색 레이아웃 패널과 흰 카드

규칙:

- 데이터 영역은 가능한 흰 배경 위에서 읽히게 한다.
- 배경 그라데이션은 전체 분위기와 브랜드 톤을 만들기 위한 용도다.
- 카드 내부에 강한 배경색을 많이 섞지 않는다.
- 강한 색 배경은 CTA, 브랜드 아이콘, 강조 배지에 제한적으로 사용한다.

## 앱 프레임

`frontend/app/(main)/layout.tsx` 기준으로 메인 앱 프레임은 다음 구조를 유지한다.

- 상단 헤더
- 중앙 스크롤 콘텐츠
- 우측 보조 사이드바
- 데스크톱 최소 폭 보장

규칙:

- 메인 업무 화면은 `min-w-[1000px]` 이상 기준으로 설계한다.
- 헤더는 얇은 chrome 역할에 집중한다.
- 사이드바는 보조 컨텍스트 패널로 사용한다.
- 새 전역 탐색 요소는 헤더와 사이드바 중 하나에만 둔다.

## 페이지 그리드

현재 주요 페이지는 카드 조합형 그리드를 쓴다.

- `markets`: 상하 3단 적층형 구조 (요약 카드 그리드 -> 시세 테이블 패널 -> 랭킹 리스트 패널)
- `stock`: 3열 대시보드 + 우측 sticky 보조 패널
- `portfolio`: 10열 기준 요약 카드 + 차트 + 상세 목록
- `signup`: 중앙 모달형 onboarding

리더보드 1~4위는 번들 순위 이미지를 작게 사용하고, 5위 이상은 숫자 배지로 표시한다. 로그인 사용자의 내 순위는 계정 정보 맥락에서만 표시한다. 헤더 상시 닉네임은 기존 밀도를 유지하고, 닉네임 클릭 팝업의 닉네임 옆과 마이페이지 계정 요약 안에 작은 배지로 배치한다.

랭커 포지션 엿보기는 markets 랭킹 패널에 붙은 오른쪽 보조 surface로 다룬다. 데스크톱에서는 랭킹 리스트를 밀어내지 않는 side popover/drawer 폭을 사용하고, locked 상태에서는 position metric 카드 자리를 비워둔 채 item count와 사용 CTA만 강조한다. unlocked 상태는 저장 snapshot card list이며 주문 패널이나 내 포지션 관리 카드와 같은 action density를 갖지 않는다.

규칙:

- 페이지는 하나의 거대한 패널보다 여러 분석 카드의 조합으로 설계한다.
- 섹션 구분은 배경색보다 카드와 간격으로 만든다.
- sticky 요소는 보조 정보와 탐색 보조에만 사용한다.
- 같은 화면에 동급 카드가 너무 많이 경쟁하지 않게 한다.

## 데스크톱 정책

현재 제품은 `only-desktop` 페이지가 있을 만큼 데스크톱 중심이다.
따라서 신규 UI도 이를 전제로 설계한다.

- 핵심 업무 화면은 데스크톱 완성도를 우선한다.
- 모바일 대응은 단순 적층보다 기능 우선순위 재정의로 접근한다.
- 복잡한 분석 그리드를 임시 대응용 세로 적층으로 망가뜨리지 않는다.

## 레이아웃 체크리스트

- 바깥 배경과 안쪽 표면이 명확히 분리되는가
- 주 콘텐츠와 보조 콘텐츠 위계가 보이는가
- sticky 요소가 본문을 가리지 않는가
- 카드 사이 간격만으로도 정보 그룹이 읽히는가

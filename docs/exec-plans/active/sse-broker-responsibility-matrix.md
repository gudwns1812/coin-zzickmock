# SSE Broker Responsibility Matrix

Status: Phase 0 inventory and regression-test handoff for the SSE broker responsibility cleanup.
Scope: backend SSE delivery classes only. This document records current ownership and the tests that pin behavior before further extraction; it does not change backend design rules.

## Phase 0 Test Inventory

- Multi-symbol market summary semantics now have explicit regression tests for the main extraction risks: one total permit per client session, per-symbol permits only for newly added symbols, release-before-register cleanup, and executor rejection telemetry.
- Unified market stream semantics now have explicit regression tests for initial snapshot failure cleanup and history-finalized fan-out targeting.
- Registry-backed candle and trading execution brokers already have targeted coverage for the common lifecycle/send mechanics that future shared-helper extraction would touch.

## Guardrails For Implementation Lanes

- Do not change external SSE endpoints, stream topology, event timing, initial snapshot behavior, or telemetry stream names without a new tested/documented decision.
- Do not introduce a generic base broker or pass-through interface.
- Common helper extraction is limited to mechanics only: `SseEmitterLifecycle` owns callback binding and completion safety; stream-specific unregister callbacks, close reasons, logs, and telemetry names stay in each broker.
- `MarketRealtimeSseBroker` now uses feature-owned concrete `MarketSummarySubscriptionRegistry` because generic `SseSubscriptionRegistry` is single-key and would complicate one-client/multi-symbol/one-emitter semantics.
